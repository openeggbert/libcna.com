Nova-3D & CNA Relationship

How CNA acts as the runtime and rendering foundation for the planned Nova-3D engine

Overview

CNA is designed not only as a standalone XNA 4.0-compatible API, but also as the internal runtime and rendering abstraction layer for Nova-3D — a planned higher-level, opinionated 3D game engine. Nova-3D will build directly on CNA's foundations, and the clean API separation between game-facing code and backend implementations is what makes this composition possible.

Rather than coupling Nova-3D to a specific graphics API from the start, the CNA layer acts as a stable, backend-agnostic interface. Nova-3D code calls CNA; CNA dispatches to EasyGL, Vulkan, bgfx, or whatever backend is active. This keeps the engine portable without Nova-3D needing to know anything about the underlying graphics stack.

The NOXNA marker found in CNA source files identifies C++ additions that are CNA- or Nova-3D-specific — things that go beyond what the original XNA 4.0 API defined. These extensions are intentionally separated from the XNA-compatible surface so that the compatibility boundary stays clear.

Architecture diagram

┌─────────────────────────────────────────────────────────┐
│                    Your Game / Nova-3D                  │
│         scene graph · ECS · editor · physics            │
└────────────────────────┬────────────────────────────────┘
                         │  calls
┌────────────────────────▼────────────────────────────────┐
│                         CNA                             │
│  GraphicsDevice · Effect · SpriteBatch · ContentManager │
│  math types · input · audio · Game loop                 │
└──────┬──────────────┬──────────────┬────────────────────┘
       │              │              │
┌──────▼──────┐ ┌─────▼─────┐ ┌────▼──────────┐
│   EasyGL    │ │   Vulkan  │ │  bgfx / SDL   │
│  (OpenGL)   │ │  backend  │ │   backends    │
└─────────────┘ └───────────┘ └───────────────┘

The diagram shows two roles for CNA: it is the target API for game code (top), and it is the abstraction boundary below which graphics backends are interchangeable (bottom). Nova-3D sits in the top layer, consuming CNA exactly as a game would.

What CNA provides

Everything a 3D engine needs at the low level is already covered by CNA:

  • GraphicsDevice abstraction — device creation, render targets, blend/depth/rasterizer states, primitive drawing
  • Effect / SpriteBatch rendering layer — stock effects (BasicEffect, SkinnedEffect, etc.), custom shader support, 2D sprite batching
  • ContentManager — asset loading pipeline for textures, models, sound effects, and effects
  • Math types — Vector2/3/4, Matrix, Quaternion, BoundingBox, BoundingSphere, Ray, and related utilities
  • Input — keyboard, mouse, gamepad, and touch input abstracted across all supported platforms
  • Audio — SoundEffect, SoundEffectInstance, MediaPlayer, and Song APIs
  • Game loop — fixed/variable timestep update, draw callbacks, and window management

What Nova-3D will add

Nova-3D is the layer that will live on top of CNA and provide the higher-level engine features that CNA intentionally does not include:

  • Scene graph — hierarchical node tree, transform inheritance, culling
  • Entity-component system (ECS) — data-oriented entity management and component composition
  • Editor tooling — visual scene editor, asset browser, inspector panels
  • Asset pipeline — offline asset processing, material authoring, level baking
  • Physics integration — collision detection and rigid-body simulation (planned via a third-party physics library)
ⓘ

Current status: Nova-3D is in the early design phase. CNA is being developed first, deliberately, so that the low-level API is stable and battle-tested before Nova-3D is built on top of it. Building the low-level layer first ensures that Nova-3D will never be inadvertently coupled to a specific graphics API or platform.

Why CNA first?

Many engines build the high-level features first and then discover that the low-level abstractions were not designed with portability in mind. By completing CNA first, Nova-3D will inherit:

  • A proven, multi-backend rendering layer (EasyGL, Vulkan, bgfx, SDL_Renderer)
  • A well-defined, XNA-compatible API surface that is already documented and tested
  • Platform support across Linux, Windows, Android, and web (Emscripten) before the engine layer is written
  • A clear boundary between engine code and backend code, making future backend additions straightforward

The NOXNA markers in the CNA source also allow Nova-3D-specific extensions to be added to the CNA layer without polluting the XNA-compatible API surface, keeping the compatibility story clean for developers who want only the XNA layer.
